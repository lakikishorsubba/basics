Basic Git Commands

What is Git?

Git is a version control system that helps you track changes in your project files and collaborate with others.

Essential Git Commands

  • Initialize repo: git init – Start a new local repository
  • Clone repo: git clone <url> – Copy a remote repo to your local machine
  • Status: git status – Show current changes and branch
  • Add files: git add <file> – Stage changes for commit
  • Commit: git commit -m "message" – Save your changes with a message
  • Push: git push origin main – Upload commits to GitHub
  • Pull: git pull – Download and merge changes from GitHub
  • Log: git log – View history of commits

Understanding Commits

A commit is like a snapshot of your project at a certain point in time. Every time you commit, Git saves what changed.

Workflow:

  1. Make changes in files
  2. git add . – Stage the changes
  3. git commit -m "Describe your change" – Save the snapshot

Good commit messages help you and your team understand the project history.

Working with Branches

A branch lets you work on a feature or fix separately from the main code (usually main).

  • git branch feature-x – Create a new branch
  • git checkout feature-x – Switch to the branch
  • git switch feature-x – (newer alternative to checkout)
  • git merge feature-x – Merge it into main
  • git branch -d feature-x – Delete the branch after merging

Branching allows safe, parallel development without affecting your main codebase.

Typical Git Workflow

  1. git clone <repo-url>
  2. git checkout -b new-feature
  3. Make changes
  4. git add .
  5. git commit -m "Added new feature"
  6. git push -u origin new-feature
  7. Create Pull Request on GitHub and merge
